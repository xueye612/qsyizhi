# 2026-05-07 H5 双端优化变更说明（相对上一版本）

## 1. 变更目标
本次版本围绕以下用户反馈进行连续优化，并按顺序完成：

1. 患者端宣教内容支持按标签筛选，点击内容后全屏阅读并自动记录阅读状态。
2. 患者端“随访”页面补充“作用说明”和更清晰的任务价值表达。
3. 患者端“我的”中长辈模式（大字模式）可稳定切换，并增强视觉反馈。
4. 医生端简化后模块不再堆到“我的”，重整到首页可直达。

---

## 2. 患者端：宣教模块升级

### 2.1 上一版本问题
- 宣教标签仅展示，不可筛选。
- 所有内容一次性平铺，不便快速定位。
- 点击内容虽然可打开，但信息层次较浅。

### 2.2 本次改造
- 新增宣教筛选状态存储键：`demo.patient.educationFilter`。
- 宣教标签改为可点击按钮：`全部 / 饮食推荐 / 用药推荐 / 运动推荐 / 注意事项`。
- 点击标签后即时过滤列表，仅展示对应分类内容。
- 保持“点击内容自动记录已读”逻辑，并在“宣教页 + 我的页”同步状态。
- 点击单条宣教内容后：
  - 以全屏弹层展示；
  - 展示标题、类型、分类、时长、来源；
  - 展示“内容摘要 + 核心要点列表”；
  - 自动标记已阅读。

### 2.3 关键文件
- `public/screens/patient-h5/app.js`

---

## 3. 患者端：随访模块优化

### 3.1 上一版本问题
- 随访页只有功能入口，用户不易理解“随访”本身价值。
- 缺少更直观的随访执行状态概览。

### 3.2 本次改造
- 顶部说明重写为“随访记录作用”导向文案。
- 新增三项概览 KPI：
  - 待处理提醒
  - 今日用药打卡
  - 症状反馈条数
- 新增“随访有什么作用？”说明卡，明确三类价值：
  - 连续判断病情
  - 减少漏服漏检
  - 提升复诊效率
- 保留原入口能力（提醒/用药/症状），但强调“统一入口、功能不合并”的策略。

### 3.3 关键文件
- `public/screens/patient-h5/app.js`

---

## 4. 患者端：长辈模式（大字模式）可用性修复

### 4.1 上一版本问题
- 虽有开关，但视觉变化不明显，用户主观上容易认为“未生效”。

### 4.2 本次改造
- 开关动作中增加即时 body class 切换，强化状态反馈。
- 在全局渲染中继续统一应用 elder-mode，避免状态不同步。
- 大字模式样式增强（显著可见）：
  - 按钮字号、最小高度上调；
  - pill、label、tag 字号上调；
  - 输入框/下拉/文本域字号与高度上调；
  - 底部 tab 文字上调；
  - KPI 数值字号上调。

### 4.3 关键文件
- `public/screens/patient-h5/app.js`
- `public/screens/patient-h5/styles.css`

---

## 5. 医生端：简化模块归位优化

### 5.1 上一版本问题
- 供体管理/科研管理/数据分析等业务模块入口集中在“我的”。
- “我的”页面信息架构偏业务化，和用户预期（个人中心）不一致。

### 5.2 本次改造
- 首页新增“业务模块直达”卡片：
  - 供体管理
  - 科研管理
  - 数据分析
- 文案明确：简化后业务模块放首页，不再放在“我的”里。
- “我的”页面收敛为个人中心：
  - 资料设置
  - 系统版本与关于
  - 退出登录
- 保留跳转逻辑（open-donors/open-research/open-analytics），但入口主位改到首页。

### 5.3 关键文件
- `public/screens/doctor-h5/app.js`
